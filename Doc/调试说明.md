# STM32F103 环境监测系统 - 调试说明

## 1. 编译与烧录

### 1.1 使用Keil MDK
1. 打开Keil MDK 5.x
2. File -> Open Project -> 选择 `STM32F103_Monitor.uvproj`
3. 点击 Build 编译（Ctrl+F7）
4. 连接ST-Link下载器
5. Debug -> Start/Stop Debug Session 进入调试
6. 点击 Download 烧录到硬件

### 1.2 常见编译错误处理
- 错误："Cannot open source file" - 检查库文件路径
- 错误："undefined reference" - 确认所有.c文件都添加到工程中

## 2. 硬件调试

### 2.1 检查清单
- [ ] 所有传感器接线正确
- [ ] 上拉电阻已连接（DHT11和I2C各需一个）
- [ ] 电源5V供电正常
- [ ] GND连接完整
- [ ] ST-Link下载器连接正确

### 2.2 OLED不显示
1. 检查I2C地址：运行I2C扫描程序确认地址
2. 检查接线：确保SCL和SDA连接到PB8和PB9
3. 检查上拉电阻：I2C需要4.7kΩ上拉电阻
4. 在OLED_Init()中调整初始化序列

### 2.3 DHT11读取异常
1. DHT11需要至少1秒的采样间隔
2. 检查PA1是否正确配置为GPIO
3. 确认4.7kΩ上拉电阻已连接
4. DHT11上电后需要等待1-2秒才能第一次读取
5. 检查是否有其他代码干扰GPIO时序

### 2.4 MQ-135数据异常
1. MQ-135需要20-30分钟预热时间
2. ADC采样时间配置为 ADC_SampleTime_239Cycles5
3. 使用分压电路时检查RL电阻值（通常为10kΩ）
4. 检查ADC是否正确配置为转换通道0

## 3. 串口调试

如需通过串口输出调试信息，可以添加USART驱动：

```c
// 配置PA9(TX) / PA10(RX)
```

使用串口助手软件查看数据：
- 波特率：115200
- 数据位：8
- 停止位：1
- 奇偶校验：无

## 4. Proteus仿真调试

### 4.1 启动仿真
1. 在Keil中编译项目生成hex文件
2. 打开Proteus设计环境
3. 在虚拟MCU中加载生成的hex文件
4. 配置各传感器仿真模型
5. 运行仿真

### 4.2 虚拟传感器配置
- DHT11：在Proteus中搜索DHT11模块
- MQ-135：使用模拟信号发生器模拟ADC输入
- OLED：使用虚拟OLED显示屏观察输出

## 5. 参数调整

### 5.1 修改采样速率
在 main.c 中修改：
```c
Delay_ms(1000);  // 改为其他值，单位毫秒
```

### 5.2 修改报警阈值
在 main.c 中修改：
```c
#define TEMP_LIMIT    30      // 温度阈值
#define HUMID_LIMIT   70      // 湿度阈值
#define GAS_LIMIT     400     // 气体浓度阈值
```

### 5.3 MQ-135校准
首次使用MQ-135时需要校准：
1. 在清洁空气中运行30分钟
2. 记录此时的RS值
3. 计算 RO = RS / 3.6  （假设在清洁空气中PPM=400）
4. 在代码中更新 MQ135_RO 值

## 6. 故障排除

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| OLED显示乱码 | I2C时序异常 | 检查I2C上拉电阻，重新初始化OLED |
| DHT11读取0值 | GPIO配置错误 | 检查PA1配置，确认为开漏输出 |
| 数据波动大 | 供电不稳定 | 添加100nF和10µF滤波电容 |
| 蜂鸣器不响 | GPIO无法输出 | 检查PA2是否配置为推挽输出 |
| 无法调试 | ST-Link连接问题 | 重新连接，更新驱动程序 |
